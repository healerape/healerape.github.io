常规介绍

  1. 按 F12 打开调试窗口
  2. 元素（Elements）- 布局和样式
  3. 控制台（Console）- js
  4. 源码（Sources）- 资源，断点调试
  5. 网络（Network）- 监控网络请求和响应
  6. 性能（Performance）- 记录和分析网页的性能数据
  7. 内存（Memory）- 内存快照
  8. 应用（Application）- 存储等
  9. 安全（Security）- 检查是否存在潜在的安全风险
  10. 审查元素工具（Inspect Element）- 选中元素
  11. 移动设备模拟器（Device Mode）- 模拟不同设备和屏幕尺寸的视图

页面元素快速调试

  1. Add Attribute: 在标签中增加新的属性

  2. Edit attribute: 编辑属性

  3. Edit as HTML: 以HTML形式更改页面元素

  4. Duplicate element: 复制元素

  5. Delete element: 删除元素

  6. Cut: 剪切

  7. Copy: 复制

  8. Paste: 粘贴

  9. Hide element： 隐藏元素

  10. Force State：强制元素状态，模拟伪类状态（如:hover、:active）以查看元素在不同状态下的样式

  11. Break on： 中断选择：设置断点，使得在选中元素上发生特定事件（如点击、DOM修改等）时暂停代码执行以便进行调试。

  12. Expand recursively 递归展开

  13. Collapse children 折叠子元素

  14. Capture node screenshot 捕获节点截图

  15. Scroll into view 使元素滚动到视图

  16. Focus 聚焦，不知道咋子用

  17. Badge settings 不知道是啥子咋子用

  18. Store as global variable 存储为全局变量，将选中元素保存为一个全局JavaScript变量，以便在控制台中使用

动态元素调试

Break on 断点调试

  1. Subtree Modification（中断子树修改）：当选择此选项时，在所选元素及其子元素发生DOM修改（添加、删除或修改子元素）时，代码执行会被暂停，使您可以查看和调试这些修改。
  2. Attributes Modification（中断属性修改）：选中此选项后，当所选元素的属性发生变化时，代码执行会被暂停。这适用于元素属性的修改，如 class、id、style 等。
  3. Node Removal（中断节点移除）：在选中此选项后，如果所选元素从 DOM 中被移除，代码执行会被暂停，允许您跟踪元素何时被移除。

监控

网络请求的监控



针对后面几个指标，它们的区别如下：

  1. Finish：
    这个指标表示页面完全加载完成所花费的时间。它包括了从开始加载到所有资源都加载完毕，所有事件都触发，以及页面的渲染和交互都准备好的时间。它代表了用户在浏览器中完整加载和交互页面所需的时间。
  2. DOMContentLoaded：
    这个指标表示从页面开始加载到DOMContentLoaded事件触发所经过的时间。DOMContentLoaded事件在DOM文档完全解析和构建后触发，这时候页面的HTML、CSS和JavaScript已经可用，可以开始进行页面的渲染和交互。通常情况下，DOMContentLoaded之前不包括外部资源的加载，如图片、样式表等。
  3. Load：
    这个指标表示从页面开始加载到Load事件触发所经过的时间。Load事件在整个页面以及所有相关资源（包括外部资源）加载完成后触发。这意味着页面的所有内容已经加载完毕，用户可以与页面进行交互。与DOMContentLoaded相比，Load事件触发后页面的渲染和交互更为完整。

监控页面重绘

作为页面动画和性能参考的重要指标之一，假设页面卡顿，可以看些哪些元素在动，作为参考
路径：右上角三个点/More tools/Rendering（渲染）

  1. Paint Flashing（绘制闪烁）：这个功能可以在页面上启用绘制闪烁模式，以帮助您可视化页面的绘制过程，识别哪些区域正在重新绘制。这有助于减少不必要的绘制操作，从而提高页面性能。
  2. Layout Shift Regions（布局变化区域）：此功能用于可视化页面布局变化的区域。布局变化可能会导致页面内容的意外移动，影响用户体验。通过分析布局变化，您可以识别和解决可能导致视觉不稳定的问题。
  3. Layer Borders（图层边框）：这个功能允许您在页面上显示图层的边界，帮助您分析页面的图层结构和合成。图层是页面中独立绘制的元素，优化图层可以提高性能。
  4. Frame Rendering Stats（帧渲染统计）：此功能显示了页面的帧渲染统计信息，帮助您了解帧率、帧间隔等性能指标。高帧率对于实现流畅的用户体验至关重要。
  5. Scrolling Performance Issues（滚动性能问题）：这个功能帮助您分析页面的滚动性能，识别可能导致滚动不流畅的问题，以及优化滚动体验。
  6. Highlight Ad Frames（高亮广告框架）：此功能可以帮助您在页面上高亮显示广告框架，从而识别哪些区域是广告内容。
  7. Core Web Vitals（核心Web性能指标）：这些是衡量用户体验的重要指标，包括Largest Contentful Paint（LCP）、First Input Delay（FID）和Cumulative Layout Shift（CLS）等。
  8. Disable Local Fonts（禁用本地字体）：这个功能允许在页面上禁用本地字体，以模拟页面使用外部字体的情况。
  9. Emulate a Focused Page（模拟聚焦页面）：这个功能用于模拟页面处于活动状态或不活动状态时的性能变化。
  10. Enable Automatic Dark Mode（启用自动暗模式）：这个功能用于模拟页面在暗模式和亮模式之间的切换。

性能分析

网络请求 js 动画等可能会导致性能问题
注意：在隐身模式下进行性能分析，才能排除插件等的干扰

  1. 打开浏览器隐身模式
  2. 打开 Perfomance 面板
  3. 右上角有个设置图标
    1. 可以设置CPU，用于模拟比较不好的电脑
    2. 设置网络等等
  4. 点击左上角进行抓取，会计算执行过程，比如执行了 n 秒后点击 stop，会有分析结果。如果通过这段时间包含了优化前和优化后的，可以进行对比。面板红色部分说明性能不好


    概念说明：FPS 帧率：60帧就代码一秒钟内有60个画面，非常流畅，反之就很卡顿

  5. 网络请求分析
    可以看出网络请求耗时、js加载 图片 css加载耗时时长

PWA

渐进式网络应用程序（Progressive Web App，PWA）是一种结合了传统网页和移动应用优点的网络应用模型。它旨在提供像移动应用一样的用户体验，包括快速加载、离线访问和推送通知等功能，同时仍然是基于Web技术开发的应用。

  • https 环境部署
  • 响应式设计，一次部署，可在移动设备和 PC 设备上运行，在不同浏览器可正常访问
  • 浏览器离线和弱网环境可极速访问
  • 可以把 App Icon 入口添加到桌面
  • 点击 Icon 入口有类似 Native App 的动画效果
  • 灵活的热更新

Service Worker

路径：Application/Service Worker
Service Worker（服务工作线程）是一种网络技术，它在网页的后台运行，与浏览器的主线程分开。它充当了可编程的代理，位于网络应用、浏览器和网络之间。Service Worker使得创建可以离线工作、快速加载并提供流畅用户体验的网络应用成为可能。它们通常用于离线缓存、推送通知和后台同步等功能。

Service Worker

  • 后台传递
  • 网络代理、转发请求、伪造响应
  • 离线缓存
  • 消息推送

Service Worker 和 Web Worker 区别

  1. 用途和功能：
    • Service Worker： 主要用于控制网络请求、缓存资源、实现离线访问和推送通知等功能。它可以在后台拦截和处理页面发出的网络请求，从而提高应用的性能和可靠性。Service Worker的设计目标是为了使Web应用在离线状态下仍能正常工作，以及提供更快速的加载和响应时间。
    • Web Worker： 用于在浏览器中创建多线程JavaScript环境，以执行计算密集型任务，如数据处理、图像处理等。Web Worker可以将这些任务从主页面线程中分离出来，以防止阻塞用户界面，从而提高页面的响应性。
  2. 运行方式：
    • Service Worker： 运行在独立的线程中，与主页面线程隔离。它可以在网页关闭或不在焦点时继续运行，从而实现后台处理、离线缓存等功能。
    • Web Worker： 同样运行在独立的线程中，但它与主页面线程之间的通信是通过消息传递进行的。Web Worker不能直接访问DOM，因此它的主要用途是执行计算任务，而不是操作页面元素。
  3. 通信机制：
    • Service Worker： 可以通过Fetch API来拦截和处理网络请求，也可以通过postMessage来与主页面进行通信。它可以在页面之间共享数据和状态。
    • Web Worker： 通信是通过postMessage进行的，主页面和Web Worker之间可以互相发送和接收消息，但消息传递会引入一些通信开销。
  4. 适用场景：
    • Service Worker： 适用于增强Web应用的性能，实现离线访问、缓存、推送通知等。它通常用于提供更好的用户体验，尤其在网络不稳定的情况下。
    • Web Worker： 适用于执行计算密集型任务，如数据处理、图像处理、复杂计算等。它主要关注的是提高页面的响应性和性能。

总的来说，Service Worker和Web Worker是两种不同的技术，用于不同的场景和目的。Service Worker主要关注应用的性能、离线访问和推送通知，而Web Worker则关注处理计算密集型任务并提高页面的响应性。

indexDB

路径：Application/Storage

  • 永久化存储
  • NoSQL 数据库
  • 测试支持

IndexedDB（Indexed Database）是一个浏览器内置的客户端数据库，用于在浏览器中存储大量结构化数据。它提供了一个异步的API，允许开发人员在客户端创建、存储和查询数据，而不需要依赖外部服务器。

以下是一些IndexedDB的基本语法和示例：

  1. 打开数据库：

    使用indexedDB.open()方法来打开或创建一个数据库。如果数据库不存在，它将被创建；如果存在，它将被打开。

    const request = indexedDB.open("myDatabase", 1);

request.onerror = event => {
  console.error("Error opening database:", event.target.errorCode);
};

request.onsuccess = event => {
  const db = event.target.result;
  console.log("Database opened successfully:", db);
};

request.onupgradeneeded = event => {
  const db = event.target.result;
  const objectStore = db.createObjectStore("myObjectStore", { keyPath: "id" });
  console.log("Database upgrade needed");
};

  2. 添加数据：

    使用事务来添加数据到对象存储（Object Store）中。

    const transaction = db.transaction(["myObjectStore"], "readwrite");
const objectStore = transaction.objectStore("myObjectStore");

const data = { id: 1, name: "John Doe", age: 30 };
const request = objectStore.add(data);

request.onsuccess = event => {
  console.log("Data added successfully");
};

  3. 查询数据：

    使用事务从对象存储中查询数据。

    const transaction = db.transaction(["myObjectStore"], "readonly");
const objectStore = transaction.objectStore("myObjectStore");

const request = objectStore.get(1);

request.onsuccess = event => {
  const result = event.target.result;
  if (result) {
    console.log("Data retrieved:", result);
  } else {
    console.log("Data not found");
  }
};

  4. 更新数据：

    使用事务更新对象存储中的数据。

    const transaction = db.transaction(["myObjectStore"], "readwrite");
const objectStore = transaction.objectStore("myObjectStore");

const data = { id: 1, name: "Jane Smith", age: 32 };
const request = objectStore.put(data);

request.onsuccess = event => {
  console.log("Data updated successfully");
};

  5. 删除数据：

    使用事务从对象存储中删除数据。

    const transaction = db.transaction(["myObjectStore"], "readwrite");
const objectStore = transaction.objectStore("myObjectStore");

const request = objectStore.delete(1);

request.onsuccess = event => {
  console.log("Data deleted successfully");
};

IndexedDB还支持索引、游标、范围查询等更高级的功能。注意，IndexedDB的API是异步的，因此在操作数据库时需要使用回调或Promise来处理结果。具体看官方文档
